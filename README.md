## Что это

**Платформа ЛКДС** — внутренний корпоративный портал, объединяющий управление задачами (замена Jira), базу знаний (замена Confluence), ИТ-поддержку и бронирование переговорок в одном приложении.

Чистый стек без фреймворков: **Node.js + Express + Vanilla JS**. Работает из коробки, без сборки и внешних баз данных.

## Возможности

### Трекер задач (ТЗ)
- Kanban-доска с drag-and-drop (нативный HTML5 API)
- Настраиваемые доски: колонки, цвета, порядок
- Swimlanes — группировка по приоритету или исполнителю
- 7 статусов: черновик → согласование → анализ → разработка → тестирование → релиз → продакшн
- Массовые операции (bulk): смена статуса/приоритета для группы задач
- Комментарии, подписки, связи между задачами
- Шаблоны ТЗ — быстрое создание из заготовок
- Умные дедлайны: автоуведомления, флаги «Просрочено» / «Скоро дедлайн»
- Excel-экспорт/импорт
- Утренний дайджест в Telegram

### База знаний (Wiki)
- Категории с иконками
- WYSIWYG-редактор (Quill v2) с загрузкой изображений
- Поиск по статьям
- Чистая типографика для чтения

### ИТ-заявки
- Пошаговый визард: категория → подкатегория → локация → описание
- Дерево: ПО, Оборудование, Сеть, ВКС, Другое
- Telegram-уведомления ИТ-отделу

### Бронирование переговорок
- 30-минутные слоты, несколько комнат
- Клик по слоту → мгновенная бронь
- Отмена полная или по слотам

### Аналитика и метрики
- Дашборд: статусы, приоритеты, типы задач
- График создания ТЗ по неделям
- Средний цикл выполнения

### Глобальный поиск
- Единый поиск по ТЗ, Wiki и ИТ-заявкам
- Быстрый доступ из любого экрана

### Безопасность
- PIN-аутентификация (scrypt-хеширование)
- Helmet CSP, HSTS, X-Frame-Options
- Rate limiting на критичные эндпоинты
- CSRF-защита через Origin
- DOMPurify для XSS-защиты
- UUID-валидация параметров

### Уведомления
- Telegram-бот: новые задачи, смена статуса, комментарии, дедлайны
- Подписка на задачу (watch) — персональные уведомления

## Архитектура

```
lkds/
├── server/
│   ├── app.js            # Express-сервер (~3000 строк)
│   └── utils.js          # Утилиты ТЗ (статусы, флаги, переходы)
├── scripts/
│   └── ai-worker.js      # AI-агент + утренний дайджест
├── public/
│   ├── index.html         # SPA точка входа
│   ├── app.js             # Фронтенд (~4100 строк)
│   ├── styles.css         # Стили (~1560 строк)
│   └── favicon.svg
├── data/                  # JSON-хранилище (auto-created)
│   ├── users.json
│   ├── tz.json
│   ├── kb-articles.json
│   ├── it-tickets.json
│   └── ...
├── docs/
│   └── ROADMAP.md         # Дорожная карта
└── .env                   # Конфигурация (из .env.example)
```

**Принципы:**
- Zero build — никакого Webpack/Vite, код работает as-is
- Zero framework — Vanilla JS, нативные Web API
- Zero DB — JSON-файлы с in-memory mutex для консистентности
- Один процесс — Express раздаёт и API, и статику

## Быстрый старт

```bash
# 1. Клонировать
git clone https://github.com/Leonid1095/lkds.git
cd lkds

# 2. Установить зависимости
npm install

# 3. Настроить окружение
cp .env.example .env
# Отредактировать .env — Telegram-токены, PIN администратора

# 4. Запустить
npm start
```

Откройте **http://localhost:3000** — портал готов к работе.

### Фоновый воркер (опционально)

```bash
npm run worker
```

AI-агент для обработки задач + утренний дайджест ТЗ в Telegram (9:00).

## Ролевая модель

| Роль | Описание |
|---|---|
| **Суперадмин** | Полный доступ: все вкладки, управление пользователями, Kanban, Wiki-редактор |
| **Админ ИТ** | Вкладка ИТ-заявки в админке |
| **Сотрудник** | Бронирование, ИТ-заявки, чтение Wiki, личный кабинет |

## Технологии

| Слой | Стек |
|---|---|
| **Сервер** | Node.js 18+, Express 4, Helmet, express-rate-limit |
| **Фронтенд** | Vanilla JS (ES2022), CSS3, HTML5 Drag & Drop |
| **Хранение** | JSON-файлы + in-memory mutex |
| **Редактор** | Quill v2 (CDN) |
| **Графики** | Chart.js 4 (CDN) |
| **Безопасность** | scrypt, DOMPurify (CDN), CSP, CSRF |
| **Уведомления** | Telegram Bot API, Nodemailer |
| **Экспорт** | ExcelJS |
