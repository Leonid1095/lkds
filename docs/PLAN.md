# План реализации MVP сайта бронирования переговорок

## Цели этапа
- Запустить простой внутренний сайт без регистрации.
- Дать прозрачную почасовую видимость занятости переговорки.
- Оставить данные контакта (ФИО + Telegram) при бронировании.
- Вынести полезные корпоративные ссылки в отдельный блок.
- Перенести подачу заявок по проблемам 1С CRM в веб-форму.
- Заложить основу для быстрого добавления новых комнат.

## Что реализуем сейчас
1. Backend на Node.js + Express с JSON-хранилищем (без БД, быстро и прозрачно).
2. API для:
   - чтения комнат,
   - чтения/создания бронирований,
   - чтения полезных ссылок,
   - создания заявок по 1С CRM.
3. Frontend (одна страница) с разделами:
   - расписание по часам (свободно/занято),
   - форма бронирования,
   - полезные ссылки,
   - форма заявки по 1С CRM.
4. Конфиг временного домена через `.env` (`PUBLIC_BASE_URL`) для легкой смены в будущем.
5. Документация по запуску и кастомизации.

## Критерии готовности
- Пользователь без регистрации видит занятые/свободные часы.
- Пользователь может создать бронь при заполнении ФИО и Telegram.
- Конфликты времени блокируются (пересечение интервалов не допускается).
- Полезные ссылки открываются из интерфейса.
- Заявка по 1С CRM сохраняется через форму.
- Добавление новой переговорки делается через файл `data/rooms.json`.
