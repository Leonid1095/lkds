# Дорожная карта развития

## Этап 1 — MVP ✅
- Почасовое расписание переговорки (свободно/занято).
- Форма бронирования с ФИО и Telegram.
- Раздел полезных ссылок.
- Веб-форма заявок по 1С CRM.

## Этап 2 — Портал сотрудника ✅ (текущий)
- Авторизация по пин-коду (сотрудник сам задаёт при регистрации).
- Профиль хранится на сервере (не зависит от браузера).
- Замена Telegram → Max (мессенджер компании).
- Бронирование на несколько часов (выбор интервала С–До).
- Клик по свободному слоту → popup записи.
- Перенос функционала crm-support-bot на сайт (ошибки/предложения 1С CRM).
- Telegram-уведомления админам при новых записях и заявках.
- Полезные ссылки в виде кнопок с иконками.
- Кнопка «Идея» — предложения по улучшению (сохранение + TG-уведомление).
- Админ-панель (отчёты): бронирования, CRM-заявки, идеи, пользователи.
- Дизайн в корпоративной палитре dxlc.ru, анимации, SVG-иконки.

## Этап 3 — Удобство и контроль
- Отмена/редактирование своих бронирований.
- Фильтры по неделе/месяцу в расписании.
- Валидация бизнес-правил (мин/макс длительность, лимит на день).
- Статусы CRM-заявок (новая → в работе → закрыта).
- Excel-экспорт отчётов из админки.
- Добавление новых переговорок через админку.

## Этап 4 — Интеграции
- Уведомления в Max (когда API станет доступен).
- SMTP-уведомления на почту (предложения, CRM-заявки).
- Вебхуки в корпоративные системы (1С, CRM).
- Синхронизация с календарями (iCal).

## Этап 5 — Масштабирование
- Переход с JSON на PostgreSQL.
- Роли и права (сотрудник / админ / суперадмин).
- История изменений и аудит действий.
- Правила бронирования для разных комнат.

## Этап 6 — Аналитика
- Дашборд загрузки переговорок (по дням/часам).
- Коэффициент утилизации и популярные слоты.
- SLA по заявкам 1С CRM.
- Статистика по сотрудникам.
